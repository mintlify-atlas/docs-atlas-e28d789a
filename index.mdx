`eslint-plugin-import` brings static analysis to your ES2015+ `import` and `export` statements. It catches broken module paths, mismatched named exports, circular dependencies, and style inconsistencies — all before your code runs.

## What it does

`eslint-plugin-import` provides 46 rules organized into four categories:

<CardGroup cols={2}>
  <Card title="Static Analysis" icon="microscope" href="/rules/no-unresolved">
    Verify that imports resolve to real files, named exports exist, and there are no circular dependencies.
  </Card>
  <Card title="Helpful Warnings" icon="triangle-exclamation" href="/rules/export">
    Detect duplicate exports, deprecated imports, extraneous dependencies, and mutable exports.
  </Card>
  <Card title="Module Systems" icon="cube" href="/rules/no-commonjs">
    Enforce or prohibit CommonJS `require`, AMD `define`, and Node.js built-in module usage.
  </Card>

## Get started

<Steps>
  <Step title="Install the plugin">
    Add `eslint-plugin-import` as a dev dependency to your project.

    ```bash
    npm install eslint-plugin-import --save-dev
    ```
  </Step>
  <Step title="Enable the recommended config">
    Extend `plugin:import/recommended` in your ESLint config to enable the most useful rules immediately.

    ```js eslint.config.js
    import importPlugin from 'eslint-plugin-import';

    export default [
      importPlugin.flatConfigs.recommended,
    ];
    ```
  </Step>
  <Step title="Add a resolver">
    Configure a resolver so the plugin can find your modules. The Node resolver is included by default.

    ```js eslint.config.js
    import importPlugin from 'eslint-plugin-import';

    export default [
      importPlugin.flatConfigs.recommended,
      {
        settings: {
          'import/resolver': {
            node: true,
          },
        },
      },
    ];
    ```
  </Step>
</Steps>

<Note>
  `eslint-plugin-import` supports ESLint v2 through v9, including the modern flat config format introduced in ESLint v9.
</Note>
