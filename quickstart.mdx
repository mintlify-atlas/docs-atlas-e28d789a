---
title: "Get started with eslint-plugin-import"
description: "Install eslint-plugin-import and configure the recommended ruleset in your project using either legacy .eslintrc or the modern flat config format."
---

`eslint-plugin-import` is an ESLint plugin, so ESLint must already be installed in your project before you begin. The steps below walk you through installing the plugin, choosing a configuration format, and verifying that linting works.

<Steps>
  <Step title="Install the plugin">
    Add `eslint-plugin-import` as a dev dependency using your package manager of choice.

    <CodeGroup>

    ```bash npm
    npm install eslint-plugin-import --save-dev
    ```

    ```bash yarn
    yarn add eslint-plugin-import --dev
    ```

    ```bash pnpm
    pnpm add eslint-plugin-import --save-dev
    ```

    </CodeGroup>

    <Note>
      The plugin supports ESLint v2 through v9. No additional peer dependencies are required for JavaScript projects. TypeScript users will also need `@typescript-eslint/parser` and `eslint-import-resolver-typescript`.
    </Note>
  </Step>

  <Step title="Configure the plugin">
    Choose the configuration format that matches your project. ESLint v8.21+ supports the flat config format (`eslint.config.js`). Older projects typically use the legacy format (`.eslintrc`).

    <Tabs>
      <Tab title="Flat config (eslint.config.js)">
        Create or update your `eslint.config.js` to extend `importPlugin.flatConfigs.recommended`:

        ```js eslint.config.js
        import importPlugin from 'eslint-plugin-import';
        import js from '@eslint/js';

        export default [
          js.configs.recommended,
          importPlugin.flatConfigs.recommended,
          {
            files: ['**/*.{js,mjs,cjs}'],
            languageOptions: {
              ecmaVersion: 'latest',
              sourceType: 'module',
            },
            rules: {
              'no-unused-vars': 'off',
              'import/no-dynamic-require': 'warn',
              'import/no-nodejs-modules': 'warn',
            },
          },
        ];
        ```
      </Tab>
      <Tab title="Legacy config (.eslintrc)">
        Extend `plugin:import/recommended` in your `.eslintrc.json` (or `.eslintrc.js` / `.eslintrc.yml`):

        ```jsonc .eslintrc.json
        {
          "extends": [
            "eslint:recommended",
            "plugin:import/recommended"
          ]
        }
        ```

        The `plugin:import/recommended` preset automatically registers the plugin and enables the recommended rules, so you do not need a separate `"plugins"` entry.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Verify the setup">
    Run ESLint against your source files to confirm the plugin is active:

    ```bash
    npx eslint src/
    ```

    To test that the plugin is catching real errors, create a file that imports something that doesn't exist:

    ```js test-import.js
    import { something } from './does-not-exist';
    ```

    Running ESLint on that file should produce an `import/no-unresolved` error:

    ```
    1:1  error  Unable to resolve path to module './does-not-exist'  import/no-unresolved
    ```

    Delete the test file once you've confirmed the plugin is working.
  </Step>
</Steps>

## What the recommended config enables

The `plugin:import/recommended` preset (and its flat config equivalent `importPlugin.flatConfigs.recommended`) enables the following rules:

| Rule | Severity | Purpose |
|------|----------|---------|
| `import/no-unresolved` | error | Ensure imports point to a file or module that can be resolved |
| `import/named` | error | Ensure named imports correspond to a named export in the remote file |
| `import/namespace` | error | Ensure imported namespaces contain the dereferenced properties |
| `import/default` | error | Ensure a default export is present, given a default import |
| `import/export` | error | Forbid invalid exports and re-exports of the same name |
| `import/no-named-as-default` | warn | Forbid use of an exported name as the identifier of a default import |
| `import/no-named-as-default-member` | warn | Forbid use of an exported name as a property of the default export |
| `import/no-duplicates` | warn | Forbid repeated imports of the same module in multiple statements |
